## 3. Information processed by the Plugin

### 3.1 Stored locally on your device (default behavior)

| Data | Purpose | Where stored |
|------|---------|--------------|
| Data source names, hosts, ports, JDBC/connection URIs | Connection management | IDE local configuration |
| Database usernames and passwords | Authenticate to your databases | IntelliJ PasswordSafe (encrypted) |
| Query history (SQL text, duration, row counts) | Developer convenience | IDE local storage |
| Plugin settings (page size, safety options, etc.) | User preferences | IDE local storage |
| Trial start timestamp | Enforce trial period | IDE local storage |

**Hanghang does not collect or receive this information unless you explicitly enable optional telemetry (see below).**

### 3.2 Optional anonymous telemetry (disabled by default)

If you enable **Settings → Database Plus → Enable anonymous telemetry**, the Plugin may send anonymized usage events such as:

- Plugin version and IDE version
- Feature usage counters (e.g., export used, schema diff opened)

Telemetry does **not** include SQL text, database row contents, hostnames, usernames, passwords, or connection strings. You may disable telemetry at any time in settings.

### 3.3 License verification (JetBrains)

Paid license validation is performed by the **IntelliJ Platform** and **JetBrains licensing infrastructure**. Hanghang does not operate a separate license server. Refer to JetBrains documentation for how IDE licensing data is processed.

## 4. Network communication

When you connect to a database, Redis, MongoDB, or other service, the Plugin communicates **directly between your IDE and the endpoints you configure**. That traffic does not pass through Hanghang servers.

## 5. What we do not collect by default

- Database table or document contents
- SQL you execute
- Credentials on Hanghang-operated servers
- Files unrelated to the Plugin

## 6. Data retention

Local Plugin data remains on your device until you delete data sources, clear history, uninstall the Plugin, or remove IDE configuration directories.

We do not retain local Plugin data on Hanghang servers because we do not receive it by default.
